We know that admin have a few high-level user needs after assigning jobs to:

  • understand how they are progressing
  • manage annotator’s workloads
  • get, analyse and quality assure the results

We decided for MVP that analysing results would be done outside of the tool. This is because:

  • we don’t know in detail how this will be done
  • we can save the results to a backend database
  • the initial admin will be a data scientist with backend access

We focused on the admin stages before results are submitted for titles.

The first part of this is a view of the status of jobs in the tool. For this, we use a table which displays:

  • a job ID, which is assigned sequentially as they are uploaded
  • the annotator name
  • the date and time it was assigned
  • the status: to do, in progress or done

These allow the annotator to get a view of progress. In workshops, we explored why they need this view. The key needs were around managing workloads. For example, if an annotator is away from work and unable to complete a job they may need to archive a job or reassign it to another annotator.

We added links to reassign or archive a job using the summary list action style.

In V1 we’ve used a table with sortable heading and filters panel from the Ministry of Justice design system. These help the user to organise the view according to the priority for managing the workload, such as in order of the date and time they were submitted, or only showing in progress jobs.

Status of titles

For an overall job, the admin can see if it is to do, in progress, or done.

As jobs could contain lots of titles, the status ‘in progress’ does not give the admin enough information to monitor workload. The job could just have been started, or be nearly complete.

We added a view of the status of tasks within a job. This uses a link in the Job ID to open a table with the list of titles within it and their status. This correlates to the annotator’s task list view.

The titles status table includes:

  • title ID
  • status

The page also shows which annotator and job number is being presented.
