We established in user needs and workflow workshops that admin may need to archive a job that’s in progress or to do. For example, if they discover a job has additional complexity that needs to be reviewed or if an annotator is unable to complete the job assigned to them and they don’t need to reassign it.

Admin users did not want to delete a job completely so there is a record of work that has already been done and they may need to activate the job again.

It was decided that archiving would be suitable at the job level – rather than individual titles. This follows the way they are assigned to annotators.

Selecting the archive job action presents a confirmation screen with inset text showing the job number and annotator name.

Confirming the archive returns to the jobs status table. A status tag for ‘archived’ is added to the job. In V1, the filters panel panel allows archived jobs to be selected. The job also keeps it’s original status tag so that admin can see if it had been started or not. If a job is archived, it only has a reassign action as it can not be archived again.
