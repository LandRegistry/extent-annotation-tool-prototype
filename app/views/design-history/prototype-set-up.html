Ahead of the project kicking off, we created some prototyping assets to use.

This included some Figma files for page templates and HMLR components so that if needed, we could create wireframes or design assets here:

  • Extent annotation tool Figma files

We used the GOV.UK prototype kit to create a blank prototype and add a layout using the HMLR design system. This is because the design system should be used for external and internal products and services, and we cannot use the gov.uk font, header or footer. The HMLR design system provides a branded header and footer and uses Arial font:

  • HMLR design system
  • Extent annotation tool prototype GitHub repository

It was decided to prototype early, as the project has a short duration and development will need to start quickly. A technical proof of concept had already been made internally which gave us something to start with.

It was decided to prototype in code so that design is more likely to be feasible working within the constraints of a small team using established design system patterns. In this case there can be a risk of developing overly complex wireframes that take more effort to build. Designing in code can help design and development by using common concepts and demonstrating relevant logic.
