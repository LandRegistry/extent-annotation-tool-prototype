As we moved into the first development sprint, it was decided for MVP that it would be more efficient to build the job status screen without using MoJ design system components which provided sortable table headers and a filters panel.

To meet the user need to find and manage jobs by different criteria in the simplest way, we designed a jobs status page using:

  • a govuk table
  • a sort panel that uses a govuk select component

In the previous version, filters allowed the user to show or hide archived jobs from the table, which would not be shown as default. This is because admin do not usually need to see archived jobs.

For this version, we added a link below the sort panel to view archived jobs. If there are no archived jobs, a page is displayed which says there are no archived jobs.

If there are archived jobs, a page is displayed with a jobs status table only showing the archived jobs. There is a link below the sort panel to return to the active jobs status page.

To allow for this additional archived jobs status page, we renamed the main jobs status page to ‘Active jobs’.

In future, we may want to add filters so that admin have more control over the data presented to quickly find certain categories of jobs.
